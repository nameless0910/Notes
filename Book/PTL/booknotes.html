1.2.3 2.2 Dreyfus 模型的五个阶段

  1. 2.2.1 新手
    1. 经验很少（经验：通过实施这项技术促进了思维的转变）
    2. 新手需要指令清单
    3. infinite regression 明确终止反复解释
    4. 规则只能让你启程，不会让你走的更远
  2. 2.2.2 高级新手
    1. 想要快速获取信息 （快速浏览文档寻找方法定义，不想寻根问底，或者温习知识）
    2. 不想要全局思维
  3. 2.2.3 胜任者
    1. 能够建立问题域的概念模型
    2. 能够解决问题
  4. 2.2.4 精通者
    1. 精通者能够自我纠正
    2. 学习他人经验
    3. 理解和运用经验之谈
    4. 知道下一步，没发生要改变什么，哪些计划要取消，应该采取什么行动
    5. 有效运用软件模式
  5. 2.2.5 专家
    1. 不断寻找更好的方法和方式去做事
    2. 专家凭直觉工作
    3. 知道哪些是无关紧要的细节，哪些是非常重要的细节

1.2.4 2.3 现实中的 Dreyfus 模型

  1. 规则断送专家
  2. 新手需要被驾驭，专家需要自己去跑
  3. 提升水平的变化
    1. 从依赖规则向依赖直觉转变
    2. 观念转变，问题不再是相关度等同的所有单元的集合体，而是一个完整和独特的个体，其中只有某些单元是相关的
    3. 从问题的旁观者变成问题涉及的系统本身的一部分
  4. 直觉和模式匹配的能力超过了显性认知

1.2.5 2.4 有效使用 Dreyfus 模型

  1. 积极的实践
    1. 需要一个明确定义的任务
    2. 任务有适当难度
    3. 任务环境可以提供大量反馈，以便于采取行动
    4. 提供重复犯错和纠正错误的机会
  2. 2.4.1 勇于承担责任
    1. 我只是执行命令是无用的
    2. 模仿榜样
    3. 没有实践就没有技能
    4. 模仿-吸收-创新
  3. 2.4.2 在实践中保持技能
    1. 专家必须持续编程，并找到一个有意义，有价值的职业生涯

1.2.6 2.5 警惕工具陷阱

  1. 模型是工具，而非镜子
  2. 混淆模型和现实
  3. 低估不能形式化的特性
  4. 规定违背个人自主性的行为
  5. 偏袒新手，从而疏远了经验丰富的员工
  6. 阐明太多细节
  7. 把复杂局势过于简单化
  8. 追求过度一致
  9. 忽视情景的细微差别
  10. 在遵从规则和自行判断之间犹豫
  11. 故弄玄虚

1.2.7 2.6 再一次考虑情境

  1. 警惕非情景化的客观性

1.2.8 2.7 日常的 Dreyfus 模型

  1. 一种规格并非处处适用
  2. 获取专业技能
    1. 培养更多的直觉
    2. 认识到情境和观察情境模式的重要性
    3. 更好的利用我们自己的经验
